#3 시스템 설계 (4단계 워크플로우)

01

Collect

모니터링 DB 기반 Naver 뉴스 API 자동 수집

기업별 검색 쿼리, Include/Exclude 키워드, 수집 주기를 반영해 최신 뉴스 후보군 확보

02

Analyze

중복·노이즈 제거 후 대표 기사 선별 및 요약

제목 유사도, 이슈 그룹, 기업명 매칭 기준으로 핵심 기사만 추려 Naver description 기반 요약 생성

03

Store

선별·요약 결과를 뉴스 DB로 구조화 및 누적 관리

기업, 주차, 대표 기사, 이벤트 유형, 요약 정보를 DB화하여 후속 조회 기반 마련

04

Dashboard

분석 DB를 웹 대시보드로 조회·검색 제공

주간 브리핑, 기업 목록, 뉴스 DB 검색, 기업별 상세 이력으로 이용자 확인 편의성 강화

#4 핵심 전략: 대상 최적화 및 토큰(비용) 절감

모니터링 대상 요약 (총 210개사, 2019년~)
  • IMPACTUPs (2019~2020): 20개사
  • ESG Korea (2021~2025): 78개사
  • True Innovation (2021~3기): 42개사
  • AI Startup Accelerator (2023~2025): 45개사
  • 창업도약패키지 (2024~2025): 25개사
2019년 이후 대상을 선정한 사유
  • 데이터 품질: 과거 데이터의 비체계성으로 인한 확인 어려움 방지
  • 전처리 과부하 방지: 불확실한 정보 기반 검색 시 필터 기준 및 키워드 제거 업무 과도화 방지
  • 모니터링 최적화: 업무 효율성과 검색 효과성 등 모니터링 관리 목적에 부합하는 대상 한정
Challenge

210개사 뉴스를 LLM으로 직접 분석 시 토큰 초과(429 에러) 발생

Solution: "깔때기형 필터링"
① 전체 검색
Naver 뉴스 API를 활용한 광범위 수집 (무료/저비용)
② 정밀 필터
파이썬 사전 마킹 로직으로 노이즈 기사 90% 제거
③ 정밀 분석
최종 선별된 '정예 기사'만 LLM 분석 적용 → 비용 및 속도 획기적 개선

#5 필터링 적용 사례

1차 필터링

검색 정밀도 향상

방법: 기업별 Include/Exclude 키워드 설정

사례: "파우스트"

  • Include: "파우스트" + 스타트업/기업 (비즈니스 맥락)
  • Exclude: 괴테, 연극, 뮤지컬, 15보병사단 등 (동음이의어 노이즈 차단)
검색 결과의 정확도를 원천적으로 향상
중복 제거

이슈 그룹화

방법: 유사 기사를 그룹화하여 최상단 대표 기사 1건만 선별 발송

사례: "콕스웨이브"

  • 이슈 그룹: 투자 유치, 해커톤 성료, 글로벌 밋업 등
  • Dedupe Keywords: 각 이슈별 핵심어(70억, OpenAI, 도쿄 등) 매칭
동일 이슈 중복 리포팅 방지 및 정보 밀도 향상
2차 필터링

시맨틱 검증

방법: 주제별 핵심 단어 검증

사례: "업스테이지"

  • 정합성 검증: 기사 제목과 본문에 반드시 포함되어야 할 필수 핵심 키워드(SK네트웍스, 영업익 등)가 제대로 일치하는지 재검증
LLM 분석 전 최종 데이터 신뢰도 확보
